# レスポンシブ・アクセシビリティ

## 1. ブレークポイント方針

特定端末名ではなく内容で崩れないようにする。

- 320pxから利用可能
- 768px前後でtablet
- 1024px以上でdesktop 3D強化
- 1440px以上でも文字幅を広げ過ぎない

## 2. モバイル優先

- モバイルではHTMLメディア中心
- 3Dは装飾のみ
- hover前提にしない
- CTAは親指で押せるサイズ
- 横スクロール禁止
- 9:16 Heroを使用

## 3. キーボード

- すべてのリンク、ボタン、メニュー、モーダルを操作可能
- focus-visibleを消さない
- Escでモーダルを閉じる
- メニュー開閉後のfocus移動を管理
- 3D装飾自体をタブ対象にしない

## 4. セマンティクス

- ページにh1は原則1つ
- セクション見出しの順序を維持
- nav, main, section, article, footerを使用
- Canvasには装飾目的の説明を設定し、重要情報を入れない

## 5. 画像・動画

- 画像にalt
- 装飾画像は空alt
- 動画内容は近くのHTML文章で説明
- 音声を含む会話動画を将来追加する場合、字幕を準備
- 自動再生Heroは無音

## 6. Motion

`prefers-reduced-motion`を最優先。

- 動画自動再生なし
- GSAP timelineを最終状態へ即時設定
- 3Dカメラ移動なし
- CSS smooth scrollなし
- 大きなズームなし

## 7. 色

- 本文と背景のコントラストを確保
- ゴールドは見出しや装飾に限定
- 状態を色だけで表現しない
- AI／REALは文字ラベルも表示

## 8. フォーム

- labelを常時表示
- errorをフィールドと関連付け
- エラー後も入力値を保持
- 成功／失敗をaria-liveで通知
- 送信中を示す

## 9. 端末性能

以下で軽量モードを選べる設計にする。

- reduced motion
- WebGL不可
- 小さい画面
- 低いdeviceMemoryが取得できる場合
- 連続フレーム低下を検出した場合は粒子を削減

低性能端末に警告を出すのではなく、自然に簡略版へ切り替える。
