# シグネチャー体験仕様(差別化の中核)

この文書は、他のモデル事務所・芸能事務所サイトが追従できない独自体験を定義する。
各演出には名前を付け、実装・レビュー時の共通語彙とする。
方針・トーンは `03_CREATIVE_AND_3D_SPEC.md`、性能条件は `11_PERFORMANCE_AND_MEDIA_LOADING.md` に従う。

実装優先度:

```text
P0(これが無いと差別化にならない): S1, S2, S4, S7
P1(入賞水準に引き上げる):        S3, S5, S6, S8
P2(余裕があれば):                S9, S10
```

すべての演出は「reduced-motion」「WebGL不可」「モバイル」の3条件で
体験の劣化ではなく“再演出”になるよう、各節にフォールバックを明記した。

---

## S1. The Awakening(プリローダー → Hero転換)

最初の3秒でブランドを確立する。

### 演出

1. 黒画面。中央に細い金の弧が現れ、読み込み進捗に応じて円(Venus Halo)を描いていく
2. 円周に沿って `LOADING — 42%` のような誌面的カウンター(セリフ体の数字)
3. 100%到達でリングが一瞬強く発光し、リングの内側から外側へ
   円形マスクが開いてHero動画が現れる(リングが「扉」になる)
4. リングはそのままHeroの3Dリングへ連続変形(位置・径を補間)
5. タグラインが行マスクで出現、CTAが続く

### 実装

- 進捗はHero poster+Hero動画metadata+主要フォントのロードを合成した擬似進捗
- 最短1.2秒・最長3.5秒でクランプ(遅い回線でも待たせすぎない)
- 2回目以降の訪問(sessionStorage)は0.6秒の短縮版
- リング→3Dリングの連続性はDOMオーバーレイとCanvasの座標同期で実現

### フォールバック

- reduced-motion: 進捗バーなしの静かなフェードイン(0.4s)
- WebGL不可: CSS conic-gradientの円弧+opacityマスク

## S2. The Meridian(実在⇄AI シェーダー転換)— サイトの中心

Hybridセクションのクライマックス。「実在の才能を、デジタルの可能性へ」を操作可能な体験にする。

### 演出

1. 画面中央に縦の金の光軸(Meridian)が走る
2. 左側=実在の世界(`hybrid-world-transform-start-01.png` / real系動画)、
   右側=デジタルの世界(`hybrid-world-transform-end-01.png` / hybrid系動画)
3. ユーザーがMeridianを左右にドラッグすると、境界がノイズを伴う
   シェーダーワイプで移動し、同一人物が実在⇄AI表現の間を行き来する
4. 境界通過時、金の微粒子が境界線から湧いて流れる
5. 放置時はスクロール進行に連動して自動で1往復デモする
6. 完了位置で `AI-ENHANCED REAL TALENT` の説明とCTAを表示

### 実装

- 1枚のPlaneに2テクスチャ+進行度uniform。ディゾルブはfbmノイズ+エッジ発光
- ドラッグはpointer events。慣性と磁気スナップ(0%/50%/100%)
- テクスチャは既存素材を使用: start/endフレームが `assets/image/source/` に存在
- 動画テクスチャ版はT3デスクトップのみ。他は静止画2枚で成立させる

### フォールバック

- モバイル: タップまたは下部スライダーで操作。ドラッグ距離を短縮
- reduced-motion: ノイズなしのクリーンな比較スライダー(機能は等価)
- WebGL不可: CSS `clip-path` のbefore/afterスライダー

## S3. Orbit Gallery(AIモデルの軌道ギャラリー)

AI Modelsセクション。名簿ではなく「星系」としてモデルを見せる。

### 演出

1. 縦長(9:16)のガラスパネル4枚+が、傾いた金の軌道リング上に配置される
2. スクロール(またはドラッグ)で軌道が回転し、正面に来たパネルがアクティブ化
3. アクティブパネルのみ動画テクスチャ再生+わずかに拡大+金の縁光
4. 非アクティブはポスター+すりガラス化(彩度・明度を落とす)
5. パネル脇にDOMでモデル名・`AI MODEL` ラベル・カテゴリーを表示
6. クリックでモデル詳細へ(S7のトランジション)

### 実装

- パネルは曲率付きPlane。軌道はスクロール進行とドラッグ量の合成角度
- モデル数はJSON駆動。4枚でも12枚でも軌道半径と間隔を自動調整(拡張性)
- 動画テクスチャは常に1本。アクティブ切替時に前をpause・解放

### フォールバック

- モバイル: 横スワイプのスナップカルーセル(CSS scroll-snap)。演出は縁光のみ
- reduced-motion: 静的グリッド+フェード
- WebGL不可: 通常のカードグリッド(MediaCard)

## S4. Kinetic Manifesto(巨大タイポの宣言)

Manifestoセクション。文章そのものを演出にする。

### 演出

1. `REAL TALENT.` `DIGITAL POSSIBILITY.` などの巨大タイポ(画面幅の80%級)が
   スクロールに連動して1行ずつマスクリビール
2. 行の一部の単語だけ金色+セリフ体に切り替わる(REAL=温かい白、AI=金)
3. スクラブで `REAL → HYBRID → AI` の語が横スライドしながら入れ替わる帯を1本入れる
4. 背景では粒子が文字の出現に同期して静かに流れる

### 実装

- SplitTextは行単位。和文は `<span>` 手動分割(禁則処理を壊さない)
- スクラブ帯はtranslateXのみ(レイアウト再計算を起こさない)

### フォールバック

- reduced-motion: 全行を最終状態で静的表示
- 演出がなくても文章として完全に読める(コピーの質で成立させる)

## S5. Atelier Wall(実在モデルの現場壁)

Real Talentセクション。「本物の現場」の質感で実在性を証明する。

### 演出

1. 撮影現場動画(`real-studio-flash-01` / `real-backstage-01`)を
   大小混在のエディトリアルグリッドで配置
2. ストロボの発光タイミングに合わせ、画面全体が一瞬だけ白く明滅(2〜4%の弱さ)
3. hoverで該当パネルが再生開始+コンタクトシート風の枠とメタ情報
   (`SHOT ON SET / TOKYO` 等の実情報のみ)が現れる
4. `REAL MODEL` ラベルはフィルムのコマ番号風にDOM表示

### 実装

- 明滅はCSS変数1つをGSAPで駆動(全要素へ波及)。発光は最大でも4%輝度
- 再生は同時1本。IntersectionObserverで管理

### フォールバック

- reduced-motion: 明滅なし。ポスター+静的枠
- 派手なグリッチは使わない(実在の証明が目的。加工感を出さない)

## S6. Constellation Index(星座ナビゲーション)

メニューを開く行為自体を体験にする。

### 演出

1. ハンバーガー(またはINDEX)でフルスクリーンの黒いオーバーレイ
2. ページ項目が金の星(点)として配置され、細い線で結ばれて星座になる
3. hoverで星が明るくなり、ページのプレビュー(ポスター)が奥に浮かぶ
4. 現在地の星はリングで囲まれる(サイト内の現在地マップを兼ねる)
5. 下部に通常のテキストリンク一覧も併記(ユーザビリティ保険)

### 実装

- 星と線はSVG(Canvasを増やさない)。粒子背景のみ既存Canvasを流用
- キーボード: Tab順はテキストリンク一覧を主経路にする

### フォールバック

- reduced-motion / モバイル: 星座は静的装飾、リンクリストが主役

---

## S7. Halo Transition(ページ遷移)

ページを「移動」ではなく「場面転換」にする。

### 演出

1. リンククリックで金のリングがクリック地点から拡大し、画面を黒で満たす
2. 遷移先ではリングが収縮しながら開いてコンテンツが現れる
3. モデルカード→詳細では、カードのポートレートが
   詳細ページのHero画像へ連続拡大(shared element)
4. 遷移時間は合計900ms以内。連打時は即時遷移に切り替え

### 実装

- View Transitions API を第一候補、非対応はDOMオーバーレイで代替
- 戻る操作(popstate)では遷移演出を半分の時間に短縮

### フォールバック

- reduced-motion: クロスフェードのみ(200ms)

---

## S8. Signature Cursor & Micro-interactions

### カーソル(fine pointerのみ)

- 金の点(6px)+120ms遅延で追従する細いリング(28px)
- メディア上: リングが拡大し `PLAY` / `DRAG` / `VIEW` ラベル
- リンク上: 磁気吸着(最大8px)+リングが要素形状に沿う
- テキスト上: 縦線カーソルに変形
- `cursor: none` はカスタムカーソン描画が保証される時のみ。タッチ環境では無効

### マイクロインタラクション

- ボタン: 金のシマー(光の帯)が1回だけ横断(hover時、600ms)
- リンク: 下線が左から右へ、離脱時は右へ抜ける(同方向に消えない)
- 画像hover: シェーダーの微細な波紋ディスプレースメント(T2以上)
- フォーム: フォーカスで金の下線が中央から伸長
- 数字(身長・件数等): ビューポート進入時にカウントアップ(セリフ体)

## S9. Ambient Audio System(P2)

- Sound On後: `venus-ambient-loop.mp3` をフェードイン(2s)
- リングの発光強度を音量のローパス成分にごく弱く連動(±5%以内)
- UI音: パネル展開・Meridian通過時のみ。-24LUFS程度の小音量
- ページ非表示(visibilitychange)でフェードアウト
- 音状態はセッション内保持。次回訪問時は再びOff

## S10. Venus Seal(エンディング)(P2)

- 最終セクション: 漂っていた金粒子が集まりロゴ(SVGパスへの引力)を結ぶ
- 結像後、リングが1周してContact CTAが浮上
- スクロールを戻すと粒子は静かに散る(可逆)
- フォールバック: ロゴSVGのstroke-dashoffset描画

---

## 実装ガードレール(全演出共通)

- 各演出は独立したコンポーネント+単一のScrollTrigger管理層に登録する
  (`SceneProgressController` に集約。個別に大量のtriggerを作らない)
- すべての演出はデータ欠落時に自動で非表示/簡略化(素材が無い状態でもビルド成功)
- 演出のON/OFFと品質は `lib/performance.ts` の品質ティア(T0〜T3)で一元制御
- 演出がユーザーの目的(モデルを見る・問い合わせる)を1秒以上遅らせない
- 各演出はStorybookまたは分離ルートで単体確認できる状態を維持する
